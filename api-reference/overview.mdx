The Dualship API allows you to programmatically manage your flows, deployments, and view execution logs.

## Base URL

All API requests should be made to:

```
https://api.dualship.run
```

## Authentication

All API endpoints require authentication using a Bearer token. Include your API key in the Authorization header:

```bash
curl -X GET "https://api.dualship.run/v1/flows" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Available Endpoints

Explore the API endpoints in the sidebar to learn how to:

- **Flows** - Create, read, update, and delete flow definitions
- **Deployments** - Deploy flows to production and manage versions
- **Executions** - View execution history and logs
- **Webhooks** - Configure webhook endpoints for your flows

## Response Format

All responses are returned in JSON format. Successful responses include the requested data, while error responses include an error message and code.

### Success Response

```json
{
  "data": {
    "id": "flow_123",
    "name": "My Flow"
  }
}
```

### Error Response

```json
{
  "error": {
    "code": "not_found",
    "message": "Flow not found"
  }
}
```

## Rate Limits

API requests are rate limited based on your plan. If you exceed the rate limit, you will receive a `429 Too Many Requests` response.
